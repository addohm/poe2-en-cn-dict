# PoE2 CN ⇄ EN Dictionary — output format

Generated by [poe2-en-cn-dict](https://github.com/). All text files are UTF-8.
`en` = International/global English, `zh` = Simplified Chinese (as shipped by the
WeGame client). Regenerate with `python update.py` after a game patch.

```
dictionary/
├─ meta.json                    run metadata, counts, validation stats
├─ lookup/
│  ├─ cn_to_en.json             Chinese → English   (best/most-frequent guess)
│  ├─ en_to_cn.json             English → Chinese   (best/most-frequent guess)
│  ├─ cn_to_en.multi.json       Chinese → [ {target, count}, ... ]  (ambiguous only)
│  └─ en_to_cn.multi.json       English → [ {target, count}, ... ]  (ambiguous only)
├─ pairs.ndjson                 every translated string with full context
└─ tables/<TableName>.json      per-table 1:1 entries, full fidelity
```

## Which file should I use?

| Need | Use |
|---|---|
| Quick term translation, one answer | `lookup/cn_to_en.json` / `lookup/en_to_cn.json` |
| All valid translations of an ambiguous term | `lookup/*.multi.json` (fall back to the `best` file if a key is absent there) |
| Translate *with* context (which table/column/id) | `pairs.ndjson` |
| Everything about one data table (items, quests, mods…) | `tables/<TableName>.json` |

### Flat lookups (`lookup/*.json`)

Simple object maps, keys sorted. Keys are outer-trimmed for clean querying;
values are the single most frequent counterpart.

```json
{ "敏捷": "Dexterity", "闪避": "Evasion", "混沌石": "Chaos Orb" }
```

Some terms are genuinely ambiguous (same word, different context). The
`.multi.json` files list every counterpart, most-frequent first. A key appears
in `.multi.json` **only if it has more than one variant**; for a complete
resolver, check `.multi.json` first and fall back to the `best` map:

```python
import json
best  = json.load(open("dictionary/lookup/en_to_cn.json", encoding="utf-8"))
multi = json.load(open("dictionary/lookup/en_to_cn.multi.json", encoding="utf-8"))

def translate(en):
    if en in multi:
        return [v["target"] for v in multi[en]]
    return [best[en]] if en in best else []
```

```js
const best  = require("./dictionary/lookup/en_to_cn.json");
const multi = require("./dictionary/lookup/en_to_cn.multi.json");
const translate = (en) => (multi[en]?.map(v => v.target)) ?? (best[en] ? [best[en]] : []);
```

### `pairs.ndjson`

One JSON object per line — the full, unaggregated record of every translated
string with its origin. Strings are **verbatim** (markup/whitespace preserved).

```json
{"table":"BaseItemTypes","column":"Name","id":"Metadata/Items/Currency/CurrencyRerollRare","index":42,"en":"Chaos Orb","zh":"混沌石"}
```

Fields: `table`, `column` (schema column name, or `__col<N>` for columns the
community schema hasn't named yet), `id` (the row's stable identifier, or `null`
for keyless tables), `index` (row number within the CN client), `en`, `zh`.

### `tables/<TableName>.json`

Per-table view, grouped by row, full fidelity. Useful when you care about a
specific data domain (e.g. `Quest.json`, `Mods.json`, `PassiveSkills.json`,
`Words.json` = unique-item names, `ClientStrings.json` = UI text).

```json
{
  "table": "Quest",
  "key": "Id",
  "columns": ["Name", "Icon_DDSFile"],
  "entries": [
    { "id": "TheIronCount", "index": 12,
      "columns": { "Name": [ { "en": "The Mad Wolf of Ogham", "zh": "欧甘的疯狼" } ] } }
  ]
}
```

- `key` — the column used as the stable row identifier (`_index` if the table
  has no unique string column).
- `columns` — the string columns that had at least one translation.
- Each `columns.<col>` is an **array** because a column can be array-typed
  (multiple strings per row); scalar columns yield a single-element array.

### `meta.json`

Run metadata: `generatedAt`, `schemaVersion`, source install paths, language
folder paths, and `stats` (entry/pair/term counts, ambiguity counts, and
`enTermsVerifiedPct` — the share of English terms confirmed present verbatim in
the international client). `perTableEntryCounts` lists entry counts per table.

## Notes for consumers

- **Non-literal / lagging CN names.** A handful of official CN translations are
  loose or slightly behind the global wording. The dictionary reflects the CN
  client exactly; it does not "fix" them.
- **Markup & placeholders.** Longer strings (mod text, tooltips, flavour) may
  contain `<default>{...}` / color tags, printf-style placeholders (`%1$s`,
  `{0}`), and newlines. These are preserved so the pair stays a faithful 1:1.
- **Not everything is a short "term".** Per the goal of covering *everything*
  one-to-one translatable, the lookups include full sentences (descriptions)
  alongside short names. Filter by length if you only want terms.
- **Unnamed columns.** Columns the community schema hasn't named appear as
  `__col<N>`; their translations are still valid, just without a friendly name.
